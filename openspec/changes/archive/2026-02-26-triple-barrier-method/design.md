## Context

在 AFML 框架的规范要求下，我们正引入由 Marcos López de Prado 提出的三重屏障法（Triple-Barrier Method, TBM）。为了彻底解决金融模型因传统的“定长窗口死扛式”打标签带来的由于路径依赖而失真的“假收益”问题，我们需要开发一套能严密复现基于持仓期间真实触发止盈和止损情境的 Label 生成器。基于性能诉求（高频数据的路径扫描运算），整体组件下沉基于 Numba 构建。

## Goals / Non-Goals

**Goals:**
- 提供原生的、基于事件（Events）以及对应价格序列的高并发三重屏障计算实现。
- 提供基于 `side` 的自动利润调改（即 Meta-Labeling 支持），把最终预测转变为二分类（是否入场）。
- 建立从 Pandas 上层建筑平滑低损耗地切入到 Numba 原生计算核心的降维降级（Downgrade）协议。
- 设计明确、完备的输出宽表（包含确切触达时刻与真实实现收益率），以承接下游样本权重模型（Sample Weights ）。

**Non-Goals:**
- 新建立或内联新的前置事件触发计算逻辑（如 CUSUM 的具体累加实现不再此承担，由上游过滤器模块计算完毕后输入）。
- 提供或打包任意的机器学习/分类器构建代码。

## Decisions

### 1. Numba Core Architecture (解耦与单核加速策略)
- **底层算法强制解耦**：核心 TBM 函数（如 `@njit()` 的底座）**完全不参与计算对数收益率或标准差阈值**。上层 `TBMLabel` 或 `FeatureKit` 计算好了完整的 `pt`（止盈）、`sl`（止损）以及目标触达序列后，仅将数值标量/张量喂入底层核心算子。
- **避免多余并发开销**：TBM 中判定“首次触达屏障（First Touch）”天生具有强烈的顺序依赖（扫描每一根后向 K 线）。我们将**不再采用 `prange` 或 `@njit(parallel=True)` 的伪并发假象**。在强串行依赖场景下，强制退行至极简的单核单线程 Numba 原生 `while/for`，可以消除额外的多线程分卷任务调度开销（Overhead），极大加快单个资产上的结算效率。

### 2. Meta-Labeling 架构与方向变量 (Side)
- 在带有 `side`（1 多头 / -1 空头）的设定下，计算回报时需要将底层收益与 `side` 相乘。
- 最初的资产三类别信号标签 (-1: 触碰止损, 0: 未触碰/超时, 1: 触碰止盈) 会通过纳入方向判断后，直接映射或坍缩为二元目标特征（1 = “根据原信号进行交易并获利”，0 = “该交易将止损或者不足盈利”）。

### 3. Data Serialization & Boundary (降级与序列化)
- **Pandas to Numpy Downgrade**: 所有的上层数据（Pandas DataFrame、DatetimeIndex）在送入 Numba 高速核前，强制提取底层 `.values`（对于日期时间，用 `.values.astype('int64')` 或 `.view('int64')` 转换为纳秒整型数字，因为 Numba 无法直接处理原生 `pd.Timestamp` 对象）。
- 这既避免了装箱/拆箱的性能惩罚，又使得内部指针迭代变得可用。

### 4. Barrier Deactivation (屏障失活规则)
- **规则设计**：如果在输入配置时，止盈乘数（`pt`）或止损乘数（`sl`）被设定为 `0` 或非数值（`NaN`），系统必须在 Numba 内对该屏障执行“失活（Deactivation）”处理。
- **处理方案**：在进入核心循环前，凡遇到 0 乘数，底层目标上/下屏障计算将统一替换为极其巨大的数值极限（如 `np.inf` 或是极大的 Float），从而使得行情在其生命周期内不可触达，实现静默剔除。

### 5. Edge Cases (Array End 数组末端或边界)
- 当发生“直至终点或截断时仍未能触及三个屏障中任何一个”的事件时：
- **强制截断或丢弃原则**：若超时屏障（`t1` 垂直时间）因序列终结未等同确立，需允许输出 `NaN` 返回时刻，并在高层对其执行针对性剔除（`dropna`）。防止模型对未完结的期末虚假价格序列进行训练拟合，造成后验泄漏。

### 6. Output Schema 规范
底层 Numba 操作结束后，必须将输出重新打包（Upload）至 Pandas 框架并统一 Schema，字段必须包含：
- 时间索引（Index）：采样事件（Event）的原始起振时间。
- `t1`：确切发生首次触及任意水平或垂直屏障的时间点戳（对齐至后向 `SampleWeights` 计算的生命周期依据）。
- `ret`：自事件发起到平仓时的该片段真实收益率（基于平仓点的收盘价/实际卖盘价）。
- `bin` 或 `label`：通过触及结果推导出的最终获胜标记类别。

## Risks / Trade-offs

- **[Risk] Numba 首次冷启动慢 (JIT 编译耗时)** -> **Mitigation**: 限制上层传递对象的动态类型，声明确凿类型的签名，并在自动化 CI 等环节环境变量开启 `$env:NUMBA_DISABLE_JIT=1` 用于快速轻量验收测试。
- **[Risk] 量纲对齐错误失效** -> **Mitigation**: 文档与 API 注释明确规定传入 `events` 的对数化阈值应当必须与对应的基础 `Close` 序列数据严格匹配形式。

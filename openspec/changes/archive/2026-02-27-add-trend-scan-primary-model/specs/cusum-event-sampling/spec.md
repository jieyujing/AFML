## ADDED Requirements

### Requirement: Flexible Event Triggering Interface
The CUSUM filtering system SHALL support an API that exposes generated `t_events` as an array-like object suitable for direct use by downstream modules like the Trend Scan regression engine.

#### Scenario: CUSUM event downstream handoff
- **WHEN** the continuous financial time series triggers the dynamic threshold
- **THEN** an event is registered and emitted as a timestamp element for downstream array indexing

### Requirement: No-Lookahead Data Consistency
The `t_events` indices generated by the CUSUM sampling module MUST rigorously align with the temporal reality of the incoming data, avoiding any future peeking that could contaminate the Backward Trend Scan calculation.

#### Scenario: Verify temporal causality
- **WHEN** an event occurs at timestamp $t_i$
- **THEN** it strictly represents volatility driven from the start of the data or last event up to exactly $t_i$

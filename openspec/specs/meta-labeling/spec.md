## ADDED Requirements

### Requirement: Side-Aware Returns Processing
The system MUST mathematically factor the directional `side` (-1/1) prediction generated by a Primary Model into the underlying threshold checking formula.

#### Scenario: Short Trade Profit
- **WHEN** `side` is -1 (Short Trade) and subsequent price drops below the defined threshold.
- **THEN** the calculated return must map as positive profit, triggering the profit-taking (pt) barrier algorithmically instead of the stop-loss (sl) barrier.

### Requirement: Meta-Label Binning Transformation
The underlying 3-class system (1, 0, -1) MUST be collapsed into a secondary binary system (1, 0) upon incorporating the `side` array, serving as the basis for bet sizing conviction estimation.

#### Scenario: Final label reduction
- **WHEN** merging the directional output of the primary classification logic with the realized performance generated by the TBM iteration.
- **THEN** the final emitted `bin` label MUST be 1 if the primary model successfully predicted the correct direction resulting in a realized profit, and 0 if the trade hit stop-loss or timed out unprofitably.

# 策略程序架构

## 1. 核心生命周期函数

### initialize(context)
**描述**：在策略开始运行时执行且仅执行一次。用于设置全局配置。
- **注意**：直接修改代码导致的变动不会在模拟盘中触发此函数，需使用 `after_code_changed`。

**参数**：
- `context`: [Context对象](../快速开始/Context对象.md)

---

### handle_data(context, data)
**描述**：单位时间（Bar）回调。
- **频率**：按天回测则每天 9:30 调用；按分钟则交易时间每分钟调用。
- **参数**：
  - `context`: Context 对象
  - `data`: 字典，key 为代码，value 为 [SecurityUnitData] 对象。

---

### run_daily / run_weekly / run_monthly (定时任务)
**描述**：指定在特定时间执行。**推荐使用此方式替代 handle_data**。

| 参数 | 类型 | 描述 |
| :--- | :--- | :--- |
| func | function | 自定义函数名，必须接受 context 参数。 |
| monthday | int | 第几个交易日。负数表示倒数。 |
| time | str | 'open', 'close', 'every_bar', 或具体时间如 '14:50'。 |
| reference_security | str | 参考标的代码，默认 '000001.XSHG'。期货建议修改。 |

---

### process_initialize(context)
**描述**：每次进程重启（模拟盘每天重启）时执行。用于初始化**不能持久化保存**的内容（如数据库查询对象）。

### after_code_changed(context)
**描述**：模拟盘更换代码后执行。用于动态修改 `g` 对象或调整参数。

## 2. 全局变量对象 g
**描述**：用于跨函数、跨时间步保存数据。
- **序列化限制**：`g` 会被 pickle 序列化保存。**不能**保存 `query` 对象、文件句柄、网络连接。
- **命名规范**：以 `__` 开头的变量（如 `g.__my_data`）不会被保存。

## 3. 策略设置函数 (必须在 initialize 中设置)

### set_benchmark(security)
- **参数**：代码字符串或 `{'000300.XSHG': 0.8, '现金': 0.2}` 权重字典。

### set_option('use_real_price', value)
- **强烈建议**：`value=True`。
- **效应**：回测时拿到的 `history` 都是基于当天日期的前复权价格，下单使用真实价格。

### set_option('order_volume_ratio', value)
- **描述**：限制单笔成交量不超过当日全市场成交量的比例（默认 1.0）。

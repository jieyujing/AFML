# 订单处理与交易控制

## 1. 下单函数全解析

### order(security, amount, style=None, side='long', pindex=0)
**描述**：买卖特定数量的标的。
- `amount`: 股数。
- `style`: `MarketOrderStyle` (市价) 或 `LimitOrderStyle(price)` (限价)。
- **注意**：下单量会自动调整为 100 的倍数（股票）。

### order_target(security, amount)
**描述**：调整至目标持仓股数。
- **示例**：`order_target('000001.XSHE', 1000)` 如果当前持有 200，则买入 800。

### order_value(security, value)
**描述**：买卖价值 `value` 元的标的。

### order_target_value(security, value)
**描述**：调整至目标持仓市值。

## 2. 撮合逻辑深度解析 ♠

聚宽的撮合引擎非常严格，以模拟真实交易：

### (1) 未启用盘口撮合 (默认)
- **买入条件**：`委托价 >= 最新价 + 滑点`。
- **卖出条件**：`委托价 <= 最新价 - 滑点`。
- **成交价**：`最新价 + 滑点` (市价单) 或 `委托价` (挂单后的限价单)。

### (2) 涨跌停限制
- **跌停**时，市价卖单会被**撤销**。
- **涨停**时，市价买单会被**撤销**。
- 限价单会进入挂单队列。

## 3. 交易成本 set_order_cost
**调用**：`set_order_cost(cost_object, type)`

**OrderCost 对象参数**：
- `open_tax`: 买入印花税。
- `close_tax`: 卖出印花税。
- `open_commission`: 买入佣金。
- `close_commission`: 卖出佣金。
- `min_commission`: 最低佣金（默认 5 元）。

---

## 4. 滑点模型 set_slippage

| 类型 | 描述 | 示例 |
| :--- | :--- | :--- |
| FixedSlippage | 固定价差 | `FixedSlippage(0.02)` |
| PriceRelatedSlippage | 百分比价差 | `PriceRelatedSlippage(0.002)` |
| StepRelatedSlippage | 跳数 (期货) | `StepRelatedSlippage(2)` |

**注意**：如果不设置，系统默认使用 `PriceRelatedSlippage(0.00246)`。

```python
def generate_paths(self, X, strategy_func, y=None, annualization_factor: float = 252.0):
    path_returns = {p: [] for p in range(self.get_n_paths())}
    for split_idx, (train_idx, test_idx, test_folds) in enumerate(self.split(X, y=y)):
        is_sharpe, fold_returns = strategy_func(train_idx, test_idx, test_folds)
        for fold_idx, _ in test_folds:
            path_idx = self._path_assignments[(split_idx, fold_idx)]
            path_returns[path_idx].extend(fold_returns.get(fold_idx, []))
    returns_paths = [pd.Series(v) for _, v in sorted(path_returns.items())]
    sharpe_paths = np.array([
        0.0 if (len(s) < 2 or s.std(ddof=1) == 0) else s.mean() / s.std(ddof=1) * np.sqrt(annualization_factor)
        for s in returns_paths
    ], dtype=np.float64)
    return returns_paths, sharpe_paths
```

## 决策与口径说明（实现时先锁定）

1. `n_splits=6, n_test_splits=2` 的路径数按组合恒等式取 `C(5,1)=5`。  
2. 规格文档出现“10 条路径”和“15 条路径”两种说法，执行时以测试契约 + 公式口径为准，并在审计报告显式记录该差异。  
3. `calculate_pbo` 主定义采用“IS 最佳策略在 OOS 排名低于中位数”的 rank 方法；旧的一维路径概率法仅作兼容入口。
